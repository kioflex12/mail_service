# mail_service — Gmail → Telegram: письма, требующие внимания

Каждое утро проверяет новую почту в Gmail, определяет моделью **Claude**, какие письма
**требуют твоего внимания** (не только работа — вакансии/офферы/рекрутёры, но и личные
сообщения, деньги, безопасность аккаунтов, дедлайны, срочные запросы), и присылает их в
**Telegram-бота**. Всё остальное (рассылки, промо, соцсети-уведомления, автошум) отсекается.

Крутится в облаке Google (Google Apps Script) — работает, даже когда твой компьютер выключен.
Ничего хостить не нужно, бесплатно.

## Что считается «важным»

| Категория | Примеры |
|-----------|---------|
| 💼 work | вакансии, офферы, рекрутёры, приглашения на собеседование, ответы с hh.ru/LinkedIn |
| ✉️ personal | личное письмо от реального человека, ждущее ответа |
| 💰 finance | счета к оплате, подозрительные списания, возвраты, важные банковские уведомления |
| 🔐 security | вход в аккаунт, смена пароля, коды 2FA, подозрительная активность |
| ⏰ deadline | дедлайны, встречи, записи на приём, напоминания к дате |
| 🚨 urgent | прямые срочные запросы, официальные/госуведомления, требующие реакции |

Границу «важно / не важно» можно подстроить — см. `system`-промпт в функции `classifyBatch_` в [Code.gs](Code.gs).

## Как это устроено

1. **Триггер по расписанию** — раз в день (по умолчанию ~09:00 Europe/Moscow).
2. **Чтение Gmail** — письма за последние 2 дня, ещё не помеченные меткой `MailServiceProcessed`.
3. **Классификация** — письма пачками уходят в Claude, тот возвращает `is_important` + категорию + причину.
4. **Отправка в Telegram** — по каждому важному письму приходит сообщение (категория, тема, отправитель, почему, ссылка «Открыть в Gmail»).
5. Разобранные письма метятся, чтобы не приходить повторно.

## Установка (≈10 минут, всё в браузере)

### 1. Telegram-бот
Бот уже есть — [@claude_emyashev_pc_bot](https://t.me/claude_emyashev_pc_bot). Открой с ним чат и
**напиши любое сообщение** (без этого бот не сможет тебе писать).

### 2. Создать проект Apps Script
- Зайди на <https://script.google.com> под нужным Google-аккаунтом → **New project**.
- Вставь содержимое [`Code.gs`](Code.gs) в файл кода (замени то, что там было).
- **Project Settings** (шестерёнка) → включи **«Show appsscript.json manifest file»**.
  Вернись в редактор, открой `appsscript.json` и вставь содержимое [`appsscript.json`](appsscript.json).

### 3. Прописать секреты (Script properties)
**Project Settings → Script properties → Add script property**, три штуки:

| Property | Значение |
|----------|----------|
| `TELEGRAM_TOKEN`   | токен бота от @BotFather |
| `ANTHROPIC_API_KEY`| ключ Claude API (console.anthropic.com) |
| `TELEGRAM_CHAT_ID` | заполним на шаге 4 |

> ⚠️ Репозиторий **публичный** — токен и ключ **никогда не вставляй в код**, только в Script properties.

### 4. Узнать свой chat_id
- В редакторе выбери функцию **`showMyChatId`** → **Run**. Дай запрошенные разрешения.
- Открой **Execution log** — увидишь строку `chat_id=...`. Скопируй число в свойство `TELEGRAM_CHAT_ID`.
- Проверь связку: запусти **`sendTestMessage`** — в Telegram должно прийти «✅ Тест…».

### 5. Включить ежедневную проверку
- Запусти функцию **`setup`** один раз. Она поставит ежедневный триггер и допросит нужные разрешения.
- Проверить работу прямо сейчас: запусти **`checkImportantMail`** вручную.

Готово — дальше оно само каждое утро.

## Настройки (вверху `Code.gs`)

| Константа | По умолчанию | Смысл |
|-----------|--------------|-------|
| `DAILY_HOUR` | `9` | час ежедневного запуска (таймзона — `timeZone` в `appsscript.json`, сейчас Europe/Moscow) |
| `LOOKBACK_DAYS` | `2` | за сколько дней смотреть письма (перекрытие для надёжности) |
| `SEARCH_QUERY_EXTRA` | `in:anywhere` | где искать; сузить до `in:inbox` или `category:primary` |
| `MODEL` | `claude-haiku-4-5` | модель-классификатор; для макс. качества → `claude-opus-4-8` / `claude-sonnet-5` |
| `MAX_THREADS` | `40` | максимум писем за запуск |

После смены `DAILY_HOUR` перезапусти `setup()` (пересоздаст триггер).

## Стоимость
- Google Apps Script — бесплатно (в рамках квот).
- Claude API — платишь только за классификацию. На Haiku 4.5 и типичном объёме входящих это копейки в месяц.

## Отладка
- **View → Executions** — история запусков и логи.
- Ждал письмо, а оно не пришло → проверь, не помечено ли оно уже меткой `MailServiceProcessed` (сними метку и запусти `checkImportantMail`), либо модель сочла его неважным.
- Приходит лишнее / не приходит нужное → подправь `system`-промпт в `classifyBatch_` (что считать/не считать важным).

```bash
npm i -g @google/clasp
clasp login
clasp clone <SCRIPT_ID>   # SCRIPT_ID из Project Settings
```
