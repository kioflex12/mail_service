# mail_service — Gmail → Telegram: важные письма с кнопками действий

Каждое утро проверяет новую почту в Gmail, определяет через **OpenRouter** (бесплатная LLM),
какие письма **требуют внимания** (работа, личные сообщения, деньги, безопасность, дедлайны,
срочное), и присылает их в **Telegram-бота** — с кнопками действий прямо под сообщением.
Рассылки, промо, соцсети-уведомления и автошум отсекаются.

Крутится в облаке Google (Google Apps Script) — работает, даже когда компьютер выключен.

## Кнопки под каждым важным письмом

| Кнопка | Что делает |
|--------|-----------|
| ✍️ **Ответ** | LLM пишет ответ → показывает в чате → **✅ Отправить** / **📝 В черновик Gmail** / **✏️ Переписать** (пришли правки текстом — перегенерит) / **✖ Отмена**. Отправка — только после твоего подтверждения. |
| 🧾 **Саммари** | сжимает длинное письмо в 2–4 пункта. |
| 📅 **В календарь** | вытаскивает дату/время и создаёт событие в Google Calendar. |
| 📥 **Архив** | убирает письмо из «Входящих» и убирает карточку из чата. |
| ✅ **Прочитано** | помечает письмо прочитанным и убирает карточку из чата. |

Долгие действия (✍️/🧾/📅 — они ходят в LLM) сначала показывают «⏳ готовлю…», затем заменяют это готовым результатом, поэтому не выглядят зависшими.

## Что считается «важным»

💼 работа · ✉️ личное · 💰 деньги · 🔐 безопасность · ⏰ дедлайны · 🚨 срочное.
Границу можно подстроить — `system`-промпт в `classifyBatch_` в [Code.gs](Code.gs).

## Как это устроено

- **Ежедневный триггер** (~09:00 МСК) читает письма за 2 дня, ещё не помеченные меткой `MailServiceProcessed`, классифицирует пачками через OpenRouter и шлёт важное в Telegram с кнопками.
- **Webhook** (`doPost`): нажатия кнопок и правки ответа прилетают на скрипт, задеплоенный как Web App. Черновик ответа между нажатиями хранится в CacheService, в кнопке — короткий id. Endpoint защищён секретом в URL (`?s=...`).
- **Ручной запуск**: команда `/check` или постоянная кнопка «🔄 Проверить почту сейчас» (появляется после `/start`) прогоняют ту же проверку по требованию и присылают короткий итог. Находят только новые, ещё не разобранные письма — как и утренний прогон.
- **Список моделей** тянется из API OpenRouter (`freeModels_`, кэш 6 ч) и не устаревает при смене `:free`-слагов; при 429/недоступности перебираются следующие.

## Установка

### 1. Telegram-бот
Создай бота через [@BotFather](https://t.me/BotFather) (`/newbot`), получи **токен**. Открой чат с ботом и **напиши ему любое сообщение**.

### 2. Ключ OpenRouter (бесплатно)
<https://openrouter.ai/keys> → **Create key** (`sk-or-v1-...`). Карта не нужна — модель по умолчанию бесплатная (`:free`).

### 3. Проект Apps Script
- <https://script.google.com/home/projects/create> под аккаунтом, **чью почту мониторим**.
- Вставь [`Code.gs`](Code.gs) в файл кода.
- **Project Settings** → включи **«Show appsscript.json manifest file»** → вставь [`appsscript.json`](appsscript.json) (в нём расширенные scopes: полный Gmail + Calendar).

### 4. Script properties
**Project Settings → Script properties**:

| Property | Значение |
|----------|----------|
| `TELEGRAM_TOKEN`     | токен бота |
| `OPENROUTER_API_KEY` | ключ OpenRouter |
| `TELEGRAM_CHAT_ID`   | твой chat_id (шаг 5) |

`WEBHOOK_SECRET` создаётся автоматически. `WEBAPP_URL` — только если понадобится (шаг 7).

> ⚠️ Репозиторий публичный — токены/ключи **только в Script properties**, не в код.

### 5. chat_id
Запусти **`showMyChatId`** → в **Execution log** будет `chat_id=...` → впиши в `TELEGRAM_CHAT_ID`. Проверка: **`sendTestMessage`** (придёт «✅ Тест…») и **`testModel`** (ответит модель).

### 6. Деплой как Web App (нужно для кнопок)
- **Deploy → New deployment** → тип **Web app**.
- **Execute as: Me**, **Who has access: Anyone** → **Deploy**.
- Пройди авторизацию — тут Google запросит новые разрешения: **отправка/изменение почты и Календарь** (это и есть кнопки ✍️/📥/✅/📅). Если покажет «app isn't verified» — это твой же скрипт: *Advanced → Go to … (unsafe) → Allow*.
- Скопируй выданный **Web app URL** (`.../exec`).

### 7. Привязать webhook
Запусти **`setupWebhook`**. В логах должно быть `"ok":true`.
Если ругнётся, что не нашёл URL — впиши скопированный `/exec` URL в Script property **`WEBAPP_URL`** и запусти `setupWebhook` снова.
Проверка: **`getWebhookInfo`** (в логах — привязанный URL, `pending_update_count`).

### 8. Включить расписание
Запусти **`setup`** (ежедневный триггер). Проверить сразу: **`checkImportantMail`**.

Готово. Каждое утро — важные письма с кнопками.

## Обновление кода потом
Меняешь `Code.gs` → **Deploy → Manage deployments → (карандаш) → Version: New version → Deploy**. Так `/exec` URL **не меняется** и webhook продолжает работать. Если создать *новый* деплой — URL сменится, тогда перезапусти `setupWebhook`.

## Настройки (вверху `Code.gs`)
| Константа | Дефолт | Смысл |
|-----------|--------|-------|
| `DAILY_HOUR` | `9` | час запуска (таймзона — `timeZone` в `appsscript.json`) |
| `LOOKBACK_DAYS` | `2` | за сколько дней смотреть письма |
| `SEARCH_QUERY_EXTRA` | `in:anywhere` | где искать (`in:inbox` / `category:primary` — уже) |
| `MODELS` | список `:free`-моделей | пробуются по порядку с фолбэком при 429/недоступности ([список](https://openrouter.ai/models?q=free)) |

## Приватность
- Содержимое писем уходит в OpenRouter (бесплатные модели могут его логировать/использовать провайдером). Для личного фильтра обычно приемлемо; нужен приват — платная модель без логирования.
- Web App исполняется «от тебя», доступ открыт всем — поэтому endpoint закрыт секретом в URL; без верного `?s=` `doPost` возвращает `forbidden`.

## Отладка
- **View → Executions** — история и логи.
- Кнопка не срабатывает → `getWebhookInfo` (виден ли URL, нет ли `last_error_message`); проверь, что деплой Web App = «Anyone» и `WEBHOOK_SECRET`/URL совпадают (перезапусти `setupWebhook`).
- `OpenRouter API 4xx` → убрали бесплатную модель или лимит на день: смени `MODEL` или подожди.
- Письмо не пришло → метка `MailServiceProcessed` (сними и запусти `checkImportantMail`) либо модель сочла неважным.
